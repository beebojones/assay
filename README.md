# assay

Harness for a blind AI model bake-off. The same control prompt goes to N
models; each returns a single-file HTML game. This repo blinds the builds so
they can be scored on one device without knowing which model made which.

N is not fixed. Add as many models as you like (up to 26 — one per letter).

## Layout

- `prompt.md` — the control prompt given identically to every model.
- `builds/` — drop raw builds here as `cosmic-pinball-<model>.html`. Contents
  are gitignored; they never go public.
- `scripts/check.py` — leak scanner. Flags vendor tells and any wrong
  `<title>`. Reports only; never edits a build.
- `scripts/shuffle.py` — blinding step. Randomizes builds to letters, copies
  them to the repo root, writes the secret key outside the repo, and
  regenerates `index.html` and `SCORECARD.md`.
- `scripts/reveal.py` — prints the letter→model mapping. Run only after
  scoring.
- `index.html`, `SCORECARD.md` — generated by `shuffle.py`. Do not hand-edit.

## Run order

1. **Drop builds** into `builds/` as `cosmic-pinball-<model>.html`
   (e.g. `cosmic-pinball-claude.html`).

2. **Scan for leaks:**

   ```
   python scripts/check.py
   ```

   Fix anything it reports. It exits nonzero if it finds a vendor tell or a
   `<title>` that isn't exactly `Cosmic Pinball`. It never strips anything for
   you.

3. **Shuffle** with a key path *outside* this repo (it refuses paths inside):

   ```
   python scripts/shuffle.py ../assay.key.json
   ```

   This re-runs the leak scan and aborts on any leak, then blinds the builds
   to `a.html`, `b.html`, … and regenerates `index.html` and `SCORECARD.md`.

4. **Commit and push** the letter files, `index.html`, and `SCORECARD.md`.
   The raw builds and the key stay out of git.

5. **Score** from the GitHub Pages URL on your phone, filling in
   `SCORECARD.md`. Letters only — you should not be able to tell which is
   which.

6. **Reveal** once scoring is done:

   ```
   python scripts/reveal.py ../assay.key.json
   ```

## Extending the vendor list

The tell list lives in `scripts/check.py` as the module-level constant
`VENDOR_TELLS`. Add new model names there as you add models.
